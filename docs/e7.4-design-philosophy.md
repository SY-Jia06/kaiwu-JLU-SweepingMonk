# E7.4 Design Philosophy

E7.4 是 early setting 下 reward 曲线最稳的模型。它的核心价值不是最终榜单成绩，而是验证了一个关键方向：确定性约束用规则接管，PPO 只学习清扫策略。

## E7.0 设计哲学

**硬编码接管的行为，从 reward 里删掉。RL 只学它该学的。**

- 充电 -> BFS 接管 -> 删充电奖励
- NPC 躲避 -> 安全层接管 -> 删 NPC 惩罚
- RL 专注：**高效清扫 + 探索新区域**

### 模型架构

分离 Actor/Critic（无共享骨干）：

- **Actor**：74D -> 128 -> 64 -> 8（动作概率）
- **Critic**：90D（74D + 16D 全局）-> 128 -> 64 -> 1（价值估计）
- 16D 全局特征 = 4 个充电桩坐标 + 4 个 NPC 坐标（/128 归一化，缺位填 -1）
- 部署时只跑 Actor，Critic 仅训练时使用（CTDE 模式）

### 奖励结构（E7.1）

| 奖励项 | 值 | 说明 |
|--------|----|------|
| 清扫 | +0.1/格 | 核心信号 |
| 步数惩罚 | -0.001 | 时间压力 |
| 重访惩罚 | -0.01 | 无清扫步惩罚（E2.1 甜蜜点）|
| 连续清扫奖励 | +0.01（>=3 连续）| 鼓励连续清扫 |
| **探索奖励** | **+0.05/首次格** | E7.1 调高（原 0.02）|
| 原地不动 | -0.05 | 连续两步同位置 |
| 电量耗尽死亡 | -5.0 | 终止惩罚 |
| NPC 碰撞死亡 | -5.0 | 终止惩罚 |
| 存活终局 | +5.0 + 5.0 * clean_ratio | 最高 +10 |
| **死亡终局** | **-3.0 + 3.0 * clean + 2.0 * survival** | E7.1 新增存活分量 |

### BFS 充电接管

- 触发：`battery <= BFS距离 + SAFETY_MARGIN(15)`
- 锁存：进入 `charge_mode` 直到电量满
- BFS 路径避开 NPC 3x3 碰撞区
- BFS 失败时回退 RL 动作（不退出 charge_mode）

### NPC 安全层

- `npc_safe_filter()`：所有动作输出后最终过滤
- 切比雪夫距离 `SAFE_DIST=3`
- 不安全时选最远离 NPC 的合法动作

## 关键洞见

### 间接奖励优于直接奖励

充电以后可以清扫更多格子，这是**间接奖励**。
删掉显式充电奖励后，RL 自己摸索到充电的价值，梯度更纯净，不会 reward hack。

### 两种存活策略都可行

模型在迭代中会发现两条路：

1. **多充电**：频繁回桩，每次满电，清扫覆盖广
2. **少充电**：找到高效路径，绕大圈，电量刚好够

两者都能活满 1000 步，路径 2 理论上时间利用率更高。
BFS 接管了“什么时候充电”，RL 学的是“去哪里清扫”——两层分工。

### finished_steps 是关键健康指标

`finished_steps / max_steps`（有效步数比例）直接反映存活质量。
E7.0 早期：579/1000 ~= 58%，说明智能体过早死亡。
调整后（E7.1）：步数趋向 900+，存活大幅改善。

### 探索奖励需要足够强

E7.0 初始探索奖励 +0.02 不足以驱动智能体离开充电桩周边安全区。
E7.1 调至 +0.05（清扫奖励的一半），探索行为明显改善。
